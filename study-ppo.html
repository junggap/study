01 // FOUNDATIONS

What is Reinforcement Learning?

Before diving into PPO, we need a firm grasp of the RL framework. An agent interacts with an environment through a loop:

🤖
AGENT
Selects actions using its policy π(a|s)
── action at ──▶
◀── state st+1, rt ──
🌍
ENVIRONMENT
Returns next state and reward signal

Core Definitions

State st

The full description of the environment at time t. Could be pixels, sensor readings, game board, etc.

Action at

What the agent does at time t. Can be discrete (left/right/jump) or continuous (torque, steering angle).

Reward rt

A scalar signal from the environment. The agent's only feedback signal — it must learn entirely from this.

Policy π(a|s)

A probability distribution over actions given state s. This is what we're optimizing.

The Goal: Maximize Return

We want to find a policy π that maximizes the expected return — the discounted sum of future rewards:

Gt = rt + γ·rt+1 + γ²·rt+2 + ... = Σk=0∞ γk · rt+k Discounted Return (γ ∈ [0,1) is the discount factor)
J(θ) = 𝔼τ~πθ [ Σt γt rt ] Objective: Expected return under policy πθ over trajectories τ

Value Functions

Two crucial quantities that estimate "how good" a state or state-action pair is:

State Value V(s)

Vπ(s) = 𝔼π[Gt | st=s]

Expected return from state s, following policy π. Answers: "How good is it to be in state s?"

Action Value Q(s,a)

Qπ(s,a) = 𝔼π[Gt | st=s, at=a]

Expected return from s, taking action a first, then following π. Answers: "How good is action a in state s?"

Advantage Function A(s,a)

The advantage is the key quantity in PPO. It answers: "How much better is action a compared to the average action in state s?"

Aπ(s, a) = Qπ(s, a) − Vπ(s) Positive advantage → action is better than average. Negative → worse than average.
// INTUITION
If A(s,a) = +2.5, it means: taking action a in state s gives 2.5 more expected return than the average action would. PPO will increase the probability of this action. If A(s,a) = -1.0, PPO will decrease it.
02 // POLICY GRADIENT THEOREM

From Calculus to Policy Gradient

We want to maximize J(θ). The natural approach: gradient ascent. We need ∇_θ J(θ). But J involves an expectation over trajectories that depend on θ — we can't differentiate through the environment.

The Log-Derivative Trick

The Policy Gradient Theorem gives us a computable gradient using the "log-derivative trick":

∇θ J(θ) = 𝔼τ~πθ [ Σt ∇θ log πθ(at|st) · Gt ] REINFORCE / Policy Gradient Theorem

Why does this work?

The key identity: ∇log f(x) = ∇f(x)/f(x), so ∇f(x) = f(x)·∇log f(x). This lets us express the gradient of an expectation as an expectation of a gradient — something we can estimate with samples from the environment.

REINFORCE Algorithm (Vanilla Policy Gradient)

Collect a trajectory

Run policy π_θ in the environment for one full episode: τ = (s₀,a₀,r₀, s₁,a₁,r₁, ...)

Compute returns Gt

For each timestep t, compute the discounted return: Gt = Σk≥0 γk rt+k

Compute gradient

g = (1/T) Σt ∇θ log πθ(at|st) · Gt

Update parameters

θ ← θ + α · g (gradient ascent since we're maximizing)

Problems with Vanilla Policy Gradient

🔴 High Variance

Returns Gt are noisy. A single lucky trajectory could push the policy far in the wrong direction. Variance grows with episode length.

🔴 Sample Inefficiency

On-policy: each batch of data can only be used for a single gradient update, then discarded. Expensive for complex environments.

🔴 Step Size Sensitivity

Too large a step → policy collapses catastrophically. Too small → unbearably slow. No principled way to set learning rate.

🔴 Non-Stationarity

Updating the policy changes the data distribution, making old gradient estimates stale immediately.

// THE CORE PROBLEM PPO SOLVES
How do we take the largest possible improvement step on the policy, without taking such a large step that we accidentally destroy performance? This is the "trust region" problem, and it's exactly what PPO addresses.
03 // COMPLETE DERIVATION — INTEGRAL TO LCLIP

From Naive Integral to PPO Objective

Every transformation, step by step, with the algebra made explicit. This is the core intellectual content of PPO.

Step 1 — The Naive Objective

We want parameters θ that maximise expected total reward. A trajectory τ = (s0,a0,r0, s1,a1,r1, …) is scored by its total discounted return R(τ) = Σt γtrt. The objective is an integral over all possible trajectories:

Objective — Naive Form
J(θ) = 𝔼τ~πθ [ R(τ) ] = ∫ pθ(τ) · R(τ) dτ
Integral over every possible trajectory, weighted by how likely πθ is to produce it, scored by its total return. Infeasible — infinite trajectory space.

Step 2 — Monte Carlo: Integral → Sample Average

An integral weighted by a probability is just an expectation. Any expectation can be estimated by sampling — this is Monte Carlo estimation:

General Monte Carlo Identity
𝔼x~p[f(x)] = ∫ p(x)·f(x) dx ≈ (1/N) Σi f(xi), xi ~ p
Apply to J(θ)
Objective — Sampled Form
J(θ) ≈ (1/N) Σi R(τi), τi ~ πθ
Run the policy N times, average their returns. The ∫dτ becomes (1/N)Σ.

Step 3 — Policy Gradient: The Log-Derivative Trick

To optimise J(θ) by gradient ascent we need ∇θJ(θ). The expectation is over a distribution pθ(τ) that depends on θ, so we can't just move ∇θ inside naively.

Step 3a — Move gradient inside integral
∇θJ(θ) = ∇θ ∫ pθ(τ)·R(τ) dτ = ∫ ∇θ pθ(τ) · R(τ) dτ
R(τ) has no θ → ∇θ acts only on pθ(τ).
Log-Derivative Identity
Key Identity: ∇f = f · ∇log f
∇θ pθ(τ) = pθ(τ) · ∇θ log pθ(τ) Proof: ∇log f = ∇f/f ⟹ ∇f = f·∇log f ✓
Substitute back
Step 3b — Gradient as Expectation
∇θJ(θ) = ∫ pθ(τ) · ∇θlog pθ(τ) · R(τ) dτ = 𝔼τ~πθ[ ∇θlog pθ(τ) · R(τ) ]
Now an expectation — we can estimate it by sampling. R(τ) is just a scalar weight. The environment needs no gradient.

Step 4 — Factorisation: Environment Terms Cancel

The trajectory probability factorises via the chain rule. Taking the log and then ∇θ, all environment terms vanish:

Trajectory Probability — Expanded
pθ(τ) = p(s0) · Πt[ πθ(at|st) · p(st+1|st,at) ] ↑ environment, no θ ↑ policy, depends on θ ↑ environment, no θ
Take log, then ∇θ — blue (env) terms → 0
Only Policy Terms Survive
∇θ log pθ(τ) = Σt ∇θ log πθ(at|st) ∴ ∇θJ(θ) = 𝔼τ~πθ[ Σt ∇θ log πθ(at|st) · R(τ) ] ≈ (1/N) Σi [ Σt ∇θ log πθ(at|st) ] · R(τi) ← REINFORCE
The environment is completely gone. We only differentiate our own policy πθ. This is why the environment doesn't need to be differentiable.

Step 5 — Reward-to-Go & Advantage

Using the full R(τ) gives credit for past rewards to future actions — causally wrong, and adds variance. Fix: weight each timestep only by its own future return, then subtract a baseline.

Problem — Full Return Ignores Causality
R(τ) = r0+…+γt-1rt-1 + γtrt+γt+1rt+1+… ^^ past: aₜ did not cause these ^^ ^^ future: aₜ causes these ^^
Replace R(τ) with reward-to-go Gt
Reward-to-Go
Gt = Σk=tT γk rk (only future rewards from t) ∇θJ(θ) = 𝔼τ~πθ[ Σt ∇θ log πθ(at|st) · Gt ]
Subtract baseline Vπ(st) — zero mean, reduces variance
Advantage Function — Final Gradient
∇θJ(θ) = 𝔼τ~πθ[ Σt ∇θ log πθ(at|st) · Ât ] where Ât = Gt − Vπ(st) Ât > 0 → action better than average → increase its probability Ât < 0 → action worse than average → decrease its probability

Step 6 — Importance Sampling: Reusing Old Data

Each update so far requires fresh samples from πθ — expensive. We want to collect data once under πθold then do multiple updates on it. Importance sampling corrects for the distribution mismatch:

Importance Sampling Identity
𝔼x~p[f(x)] = 𝔼x~q[ p(x)/q(x) · f(x) ] Proof: ∫ q(x)·(p(x)/q(x))·f(x) dx = ∫ p(x)·f(x) dx ✓
Apply: p = πθ, q = πθold
Gradient Rewritten Under Old Policy
𝔼τ~πθ[ Σt ∇θlog πθ(at|st) · Ât ] = 𝔼τ~πθold[ πθ(at|st) / πθold(at|st) · ∇θlog πθ(at|st) · Ât ]
Now the expectation is under πθold — we can use our stored rollouts.
Simplify: (p/q)·∇log p = ∇(p/q)
The Algebraic Collapse
πθ/πθold · ∇θlog πθ = πθ/πθold · ∇θπθ/πθ (since ∇log f = ∇f/f) = (1/πθold) · ∇θπθ = ∇θ( πθ/πθold ) = ∇θ r(θ) where r(θ) = πθ(at|st) / πθold(at|st)
Pull ∇θ outside → defines surrogate objective
Surrogate Objective LCPI
∇θJ(θ) = ∇θ 𝔼τ~πθold[ r(θ) · Ât ] Maximise: LCPI(θ) = 𝔼t[ r(θ) · Ât ] where r(θ) = πθ(at|st) / πθold(at|st) r(θ) = 1 when θ = θold r(θ) > 1 new policy more likely to take this action r(θ) < 1 new policy less likely
Data reuse unlocked — but unconstrained LCPI lets r(θ) grow arbitrarily large, making importance weights inaccurate and destroying performance. We need to constrain how far r(θ) moves.

Step 7 — The PPO Clip: Constraining r(θ)

TRPO constrains via a KL divergence penalty — correct but requires expensive second-order optimisation. PPO's insight: just clip r(θ) directly.

TRPO Constraint (too expensive)
maximise 𝔼t[ r(θ) · Ât ] subject to 𝔼t[ KL[ πθold(·|st) ‖ πθ(·|st) ] ] ≤ δ Requires conjugate gradient + Hessian-vector products — O(n²) cost
PPO's simpler alternative
Clipped Ratio
clip( r(θ), 1−ε, 1+ε ) = 1−ε if r(θ) < 1−ε ← floor: prevent too much decrease r(θ) if 1−ε ≤ r(θ) ≤ 1+ε ← free zone: gradient flows 1+ε if r(θ) > 1+ε ← ceiling: prevent too much increase
Take min of clipped and unclipped — pessimistic lower bound
★ PPO-Clip Objective — Final Form
LCLIP(θ) = 𝔼t[ min( r(θ)·Ât, clip(r(θ), 1−ε, 1+ε)·Ât ) ] Why min()? It gives the worst-case (most conservative) estimate: Ât > 0, r > 1+ε → clipped term smaller → min picks it → no gradient Ât > 0, r < 1+ε → unclipped smaller → min picks it → gradient flows Ât < 0, r < 1−ε → clipped term larger → min picks it → no gradient Ât < 0, r > 1−ε → unclipped larger → min picks it → gradient flows
ε = 0.2 is the standard value. r(θ) is computed in log-space as exp(log πθ − log πθold) for numerical stability.
// DERIVATION SUMMARY
Infeasible integral → Monte Carlo sampling → log-derivative trick → gradient as expectation → factorisation removes environment → reward-to-go fixes causality → advantage reduces variance → importance sampling enables data reuse → clip prevents trust-region violation = LCLIP
04 // PPO-CLIP OBJECTIVE — DEEP DIVE

The Clipping Mechanism

We derived LCLIP in the previous section. Here we go deeper — intuition, interactive visualisation, and the case-by-case analysis of when clipping engages.

LCLIP(θ) = 𝔼t[ min( rt(θ)·Ât , clip(rt(θ), 1−ε, 1+ε)·Ât ) ] PPO-Clip Objective — ε is typically 0.1 or 0.2

Unpacking the Formula

There are two terms inside the min():

Term 1: rt(θ)·Ât

The unclipped surrogate objective (same as TRPO's objective). This is what we'd optimize without any constraint.

Term 2: clip(rt(θ), 1−ε, 1+ε)·Ât

The clipped version: ratio is forced to stay within [1−ε, 1+ε]. Any change beyond this boundary is ignored.

Case Analysis: When Does Clipping Matter?

Advantage ÂtRatio r(θ)Active TermEffect
Positive (+)r < 1+εUnclippedGradient flows: increase action prob
Positive (+)r > 1+εClippedNo gradient: we've increased enough
Negative (−)r > 1−εUnclippedGradient flows: decrease action prob
Negative (−)r < 1−εClippedNo gradient: we've decreased enough
// THE GENIUS OF MIN()
Taking the minimum of the clipped and unclipped objectives creates a pessimistic lower bound. We only get credit for the worse of the two estimates — this prevents the policy from being overly optimistic about changes outside the trust region.

PPO-KL Variant (Alternative)

An alternative to clipping uses an adaptive KL penalty:

LKLPEN(θ) = 𝔼t[ rt(θ)·Ât − β·KL[πθold‖πθ] ] PPO-KL: β is adapted based on measured KL (increase if KL too large, decrease if too small)

In practice, PPO-Clip is used more often because it's simpler, more robust, and doesn't require tuning the adaptive KL schedule.

05 // GENERALIZED ADVANTAGE ESTIMATION

Computing the Advantage: GAE

We need estimates of Ât. The quality of these estimates massively affects training stability. Generalized Advantage Estimation (GAE) provides a principled way to do this.

The Bias-Variance Tradeoff

1-step TD Error δt

δt = rt + γV(st+1) − V(st)

Low variance (single bootstrap). High bias (depends on V accuracy).

Monte Carlo Return

Ât = Gt − V(st)

Zero bias (actual rollout). High variance (many random steps accumulate).

GAE interpolates between these two extremes using a parameter λ ∈ [0,1]:

ÂtGAE(γ,λ) = Σl=0∞ (γλ)l · δt+l GAE: exponentially-weighted sum of TD errors. λ→0: 1-step TD. λ→1: Monte Carlo.

This has a beautiful recursive form for efficient computation:

Ât = δt + γλ · Ât+1 Recursive GAE — computed backwards from the end of the trajectory

Typical Hyperparameters

γ = 0.99

Discount factor. High γ considers far future. Lower γ makes the agent myopic but reduces variance.

λ = 0.95

GAE parameter. Close to 1 → less bias, more variance. Close to 0 → more bias, less variance.

06 // FULL PPO ALGORITHM

Putting It All Together

The Complete PPO Objective

PPO optimizes a combined loss with three components:

LPPO(θ) = 𝔼t[ LCLIP(θ) − c1·LVF(θ) + c2·S[πθ](st) ] Full PPO objective. c1 ≈ 0.5 (value coeff), c2 ≈ 0.01 (entropy coeff)

L^CLIP — Policy Loss

The clipped surrogate objective. Maximizing this improves the policy while staying within the trust region.

L^VF — Value Loss

LVF = (Vθ(st) − Vttarg)²

MSE between predicted value and target value. Trains the critic to accurately predict returns.

S — Entropy Bonus

S[πθ](s) = −Σa π(a|s) log π(a|s)

Encourages exploration by preventing the policy from becoming too deterministic too quickly.

// WHY SHARED BACKBONE?
The backbone learns general representations useful for both the policy and value estimation. Gradients from both LCLIP and LVF flow through it simultaneously — the backbone is being trained to extract features that help predict good actions and accurate values at the same time. The value coefficient c1 balances how much each loss influences the shared weights.
// SEPARATE vs SHARED NETWORKS
With a shared backbone: set c1 = 0.5. Gradients from both heads fight over shared weights — this can cause instability if value loss is much larger than policy loss.

With separate networks: set c1 = 1.0, run two separate optimisers. More stable but uses more memory and compute. Common in robotics and high-dimensional tasks.

PPO Algorithm: Step by Step

Collect Rollouts

Run current policy πθold in parallel environments for T timesteps. Collect (st, at, rt, st+1, donet) for each step. Store log probabilities log πθold(at|st).

Compute Advantages and Returns

Use the critic Vθ to compute TD errors δt. Apply GAE backwards to get Ât. Compute value targets: Vttarg = Ât + Vθ(st).

Normalize Advantages

Normalize advantages across the batch: Â ← (Â − mean(Â)) / (std(Â) + ε). This stabilizes training and makes the gradient scale-invariant.

K Epochs of Minibatch Updates

For K epochs (typically 4–10), shuffle the rollout buffer, split into minibatches of size M. For each minibatch: compute current log probs, ratios r(θ), clip, compute full loss LPPO, backpropagate, clip gradients, update θ.

Update Old Policy

Set θold ← θ. Discard the rollout buffer (on-policy). Repeat from step 1.

1. Actor-Critic Network

PYTHON
import torch
import torch.nn as nn
import torch.nn.functional as F
from torch.distributions import Categorical, Normal
import numpy as np

class ActorCritic(nn.Module):
    """Shared-backbone actor-critic for PPO."""
    
    def __init__(self, obs_dim: int, act_dim: int,
                 hidden: int = 64, continuous: bool = False):
        super().__init__()
        self.continuous = continuous
        
        # Shared feature extractor (orthogonal init is key!)
        self.backbone = nn.Sequential(
            nn.Linear(obs_dim, hidden),  nn.Tanh(),
            nn.Linear(hidden, hidden),   nn.Tanh(),
        )
        
        # Actor head: outputs action distribution parameters
        self.actor = nn.Linear(hidden, act_dim)
        
        # Critic head: outputs scalar state value
        self.critic = nn.Linear(hidden, 1)
        
        # For continuous actions: learnable log_std
        if continuous:
            self.log_std = nn.Parameter(torch.zeros(act_dim))
        
        # Orthogonal initialization (crucial for PPO stability)
        self._init_weights()
    
    def _init_weights(self):
        for m in self.backbone:
            if isinstance(m, nn.Linear):
                nn.init.orthogonal_(m.weight, gain=np.sqrt(2))
                nn.init.constant_(m.bias, 0.0)
        # Actor: small init so policy starts near uniform
        nn.init.orthogonal_(self.actor.weight, gain=0.01)
        # Critic: standard init
        nn.init.orthogonal_(self.critic.weight, gain=1.0)
    
    def get_value(self, obs):
        return self.critic(self.backbone(obs))
    
    def get_action_and_value(self, obs, action=None):
        features = self.backbone(obs)
        value = self.critic(features)
        
        if self.continuous:
            mu = self.actor(features)
            std = self.log_std.exp().expand_as(mu)
            dist = Normal(mu, std)
        else:
            logits = self.actor(features)
            dist = Categorical(logits=logits)
        
        if action is None:
            action = dist.sample()
        
        log_prob = dist.log_prob(action)
        entropy = dist.entropy()
        
        return action, log_prob, entropy, value

2. Rollout Buffer

PYTHON
class RolloutBuffer:
    """Stores on-policy experience for PPO updates."""
    
    def __init__(self, T: int, n_envs: int, obs_dim: int,
                 act_dim: int, gamma: float, lam: float):
        self.T, self.n_envs = T, n_envs
        self.gamma, self.lam = gamma, lam
        
        # Pre-allocate tensors for efficiency
        self.obs      = torch.zeros(T, n_envs, obs_dim)
        self.actions  = torch.zeros(T, n_envs, act_dim)
        self.rewards  = torch.zeros(T, n_envs)
        self.dones    = torch.zeros(T, n_envs)
        self.values   = torch.zeros(T, n_envs)
        self.log_probs= torch.zeros(T, n_envs)
        self.advantages = torch.zeros(T, n_envs)
        self.returns  = torch.zeros(T, n_envs)
        self.ptr = 0
    
    def store(self, obs, act, rew, done, val, logp):
        self.obs[self.ptr] = obs
        self.actions[self.ptr] = act
        self.rewards[self.ptr] = rew
        self.dones[self.ptr] = done
        self.values[self.ptr] = val
        self.log_probs[self.ptr] = logp
        self.ptr += 1
    
    def compute_gae(self, last_value, last_done):
        """Compute GAE advantages and value targets."""
        gae = 0
        for t in reversed(range(self.T)):
            if t == self.T - 1:
                next_non_terminal = 1.0 - last_done
                next_value = last_value
            else:
                next_non_terminal = 1.0 - self.dones[t + 1]
                next_value = self.values[t + 1]
            
            # TD error: δₜ = rₜ + γ·V(sₜ₊₁) − V(sₜ)
            delta = (self.rewards[t]
                     + self.gamma * next_value * next_non_terminal
                     - self.values[t])
            
            # Âₜ = δₜ + γλ·Âₜ₊₁  (recursive GAE)
            gae = delta + self.gamma * self.lam * next_non_terminal * gae
            self.advantages[t] = gae
        
        # Value targets for critic training
        self.returns = self.advantages + self.values
    
    def get_batches(self, batch_size: int):
        """Flatten buffer and yield random minibatches."""
        B = self.T * self.n_envs
        
        # Normalize advantages (batch-level)
        adv = self.advantages.reshape(-1)
        adv = (adv - adv.mean()) / (adv.std() + 1e-8)
        
        idx = torch.randperm(B)
        for start in range(0, B, batch_size):
            mb = idx[start : start + batch_size]
            yield (
                self.obs.reshape(B, -1)[mb],
                self.actions.reshape(B, -1)[mb],
                self.log_probs.reshape(-1)[mb],
                adv[mb],
                self.returns.reshape(-1)[mb],
            )

3. PPO Update Step

PYTHON
def ppo_update(
    agent: ActorCritic,
    buffer: RolloutBuffer,
    optimizer: torch.optim.Optimizer,
    clip_eps: float = 0.2,
    value_coef: float = 0.5,
    entropy_coef: float = 0.01,
    max_grad_norm: float = 0.5,
    n_epochs: int = 10,
    batch_size: int = 64,
):
    stats = {'policy_loss': [], 'value_loss': [], 
             'entropy': [], 'clip_frac': [], 'approx_kl': []}
    
    for epoch in range(n_epochs):
        for obs, acts, old_logp, advs, returns in buffer.get_batches(batch_size):
            
            # Forward pass with CURRENT policy
            _, new_logp, entropy, new_value = agent.get_action_and_value(obs, acts)
            new_value = new_value.squeeze(-1)
            
            # ── PROBABILITY RATIO ──
            # r(θ) = π_θ(a|s) / π_θ_old(a|s) = exp(log π_θ − log π_θ_old)
            log_ratio = new_logp - old_logp
            ratio = log_ratio.exp()
            
            # ── CLIP OBJECTIVE ──
            surr1 = ratio * advs                             # unclipped
            surr2 = torch.clamp(ratio, 1-clip_eps, 1+clip_eps) * advs  # clipped
            policy_loss = -torch.min(surr1, surr2).mean()  # negative: we maximize
            
            # ── VALUE LOSS (optionally clipped) ──
            value_loss = F.mse_loss(new_value, returns)
            
            # ── ENTROPY BONUS ── (maximized → subtract from loss)
            entropy_loss = -entropy.mean()
            
            # ── COMBINED LOSS ──
            loss = policy_loss + value_coef * value_loss + entropy_coef * entropy_loss
            
            # ── BACKPROP + GRADIENT CLIP ──
            optimizer.zero_grad()
            loss.backward()
            nn.utils.clip_grad_norm_(agent.parameters(), max_grad_norm)
            optimizer.step()
            
            # Diagnostics
            with torch.no_grad():
                approx_kl = ((ratio - 1) - log_ratio).mean().item()
                clip_frac = (torch.abs(ratio - 1) > clip_eps).float().mean().item()
            
            stats['policy_loss'].append(policy_loss.item())
            stats['value_loss'].append(value_loss.item())
            stats['entropy'].append(-entropy_loss.item())
            stats['approx_kl'].append(approx_kl)
            stats['clip_frac'].append(clip_frac)
    
    return {k: np.mean(v) for k, v in stats.items()}

def train_ppo(env_fn, total_timesteps=1_000_000):
    # Hyperparameters
    N_ENVS   = 8      # parallel environments
    T        = 2048   # steps per rollout per env
    EPOCHS   = 10     # update epochs per rollout
    BATCH    = 64     # minibatch size
    LR       = 3e-4   # learning rate (often annealed)
    GAMMA    = 0.99   # discount factor
    LAM      = 0.95   # GAE lambda
    CLIP_EPS = 0.2    # clip epsilon
    
    envs   = VecEnv(env_fn, N_ENVS)
    agent  = ActorCritic(obs_dim, act_dim)
    optim  = torch.optim.Adam(agent.parameters(), lr=LR, eps=1e-5)
    buffer = RolloutBuffer(T, N_ENVS, obs_dim, act_dim, GAMMA, LAM)
    
    obs = envs.reset()
    n_updates = total_timesteps // (T * N_ENVS)
    
    for update in range(n_updates):
        
        # ── OPTIONAL: Linear LR annealing ──
        frac = 1.0 - update / n_updates
        optim.param_groups[0]['lr'] = LR * frac
        
        # ── STEP 1: Collect rollouts ──
        buffer.ptr = 0
        for t in range(T):
            with torch.no_grad():
                action, logp, _, value = agent.get_action_and_value(obs)
            next_obs, reward, done, info = envs.step(action)
            buffer.store(obs, action, reward, done, value, logp)
            obs = next_obs
        
        # ── STEP 2: Compute GAE ──
        with torch.no_grad():
            last_val = agent.get_value(obs)
        buffer.compute_gae(last_val, done)
        
        # ── STEP 3: PPO Update ──
        stats = ppo_update(agent, buffer, optim,
                            clip_eps=CLIP_EPS, n_epochs=EPOCHS, batch_size=BATCH)
        
        # Log every 10 updates
        if update % 10 == 0:
            print(f"Update {update} | "
                  f"Policy: {stats['policy_loss']:.3f} | "
                  f"Value: {stats['value_loss']:.3f} | "
                  f"KL: {stats['approx_kl']:.4f} | "
                  f"Clip%: {stats['clip_frac']:.2f}")
08 // TIPS & TRICKS

Making PPO Work in Practice

Critical Implementation Details

Observation Normalization

Normalize observations using a running mean and variance. This is often the most important trick. Use a running normalization wrapper that updates during rollouts.

# Running normalization
obs = (obs - obs_rms.mean) / (np.sqrt(obs_rms.var) + 1e-8)
obs = np.clip(obs, -10, 10)  # clip to prevent extreme values

Reward Normalization/Scaling

Normalize rewards using a running estimate of the return's standard deviation. Or simply scale rewards to a known range. Without this, value learning is unstable.

Gradient Clipping

Always clip gradients by global norm (max_grad_norm=0.5). This prevents catastrophic updates when advantages are large.

Learning Rate Annealing

Linearly decay learning rate from initial value to 0 over training. Helps fine-tune the policy in later stages when it's close to optimal.

Clipped Value Loss

Optionally clip the value function update too (similar to policy clip). Prevents the value function from making large updates:

v_clipped = old_val + torch.clamp(new_val - old_val, -eps, eps)
v_loss1 = (new_val - returns).pow(2)
v_loss2 = (v_clipped - returns).pow(2)
value_loss = torch.max(v_loss1, v_loss2).mean()

Early Stopping on KL

Stop update epochs early if approximate KL divergence exceeds a threshold (e.g., 0.015). This prevents over-optimizing on stale data:

if approx_kl > 1.5 * target_kl:
    break  # stop epochs early

Common Failure Modes

🔴 KL explodes

Early KL > 0.05 constantly. Fix: lower learning rate, add early stopping, check advantage normalization.

🔴 Policy entropy → 0

Policy collapses to a single action immediately. Fix: increase entropy coefficient, check reward scaling.

🔴 Value loss diverges

NaN in training. Fix: normalize observations and rewards, lower learning rate, add gradient clipping.

🔴 No learning

Flat reward curve. Check: environment reward scaling, γ appropriate for horizon, policy can actually explore.

// WHEN TO USE PPO
PPO is the default choice for most RL problems: it's stable, robust to hyperparameters, scales well with parallel environments, and works for both discrete and continuous action spaces. Use SAC/TD3 when sample efficiency is critical (real-world robotics). Use DQN for discrete actions with large replay buffers. PPO was used in OpenAI Five (Dota 2), ChatGPT RLHF, and most robotics locomotion benchmarks.
